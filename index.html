🎲 Provably-Fair Roll Verifier

Recompute any roll yourself, right here in your browser. Nothing is sent anywhere — this page has no server and no outside scripts. Right-click → View Source to confirm.

Each roll is HMAC-SHA512(server seed, "client seed-nonce"). The server seed was committed (its SHA-256 shown) before you played, so it couldn't be changed after the fact. Your client seed is yours. So neither side can rig the outcome.

Tip: server seed + client seed stay the same for a whole session — bump the nonce and re-verify to check every roll in that session.

How does this prove the game is fair?
  1. Before you play, the bot shows you the SHA-256 hash of its secret server seed (,serverseed). A hash can't be reversed, so you learn nothing about the seed — but the bot is now locked in.
  2. You set your own client seed (,clientseed). The bot can't predict or change it.
  3. Every roll is HMAC-SHA512(server seed, "{client seed}-{nonce}"). The nonce counts up by one each roll.
  4. The first 13 hex characters of that hash become a number from 0 to 1 (÷ 2^52), then scaled into the roll's range.
  5. Afterwards the bot reveals the server seed (,verify). You hash it here — if it matches the commitment from step 1 and every roll recomputes to what you saw, the bot didn't cheat. It couldn't have, without breaking SHA-512.
